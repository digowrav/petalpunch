How Minimax Works

The AI models the game as a tree of all possible futures. MAX (you) tries to maximize the score. MIN (AI) tries to minimize it. Alpha-beta pruning skips branches that can't affect the final decision.

What's This?
🔍
Nodes Explored
The number of possible future board states the AI evaluated before making its move. Each "node" is one possible arrangement of the board that the AI considered.
Example: If nodes = 5,631, the AI looked at 5,631 different possible game positions to find the best move. More nodes = the AI thought harder.
✂
Pruned
The number of branches the AI skipped because it couldn't possibly lead to a better outcome. This is alpha-beta pruning, a smart shortcut that gets the same perfect answer while doing less work.
Example: If pruned = 2,138, the AI was smart enough to skip 2,138 game states it didn't need to check. Higher pruning = more efficient search.
📏
Depth
How many moves ahead the AI looked. A depth of 9 means it searched all the way to the end of every possible game — it can see the entire future. A depth of 3 means it only looked 3 moves ahead.
Easy (Q-learning): depth 0 — uses learned experience, not search
Medium (minimax): depth 3 — looks 3 moves ahead
Hard (α-β pruning): depth 9 — sees everything
Why do these numbers change? Each difficulty uses a different AI algorithm. Easy mode uses Q-learning (a neural-network-like approach that learned from playing 100,000+ games against itself). Medium uses minimax with limited depth. Hard uses full minimax with alpha-beta pruning (it plays perfectly and can never lose).
